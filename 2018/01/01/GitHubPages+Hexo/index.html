利用GitHubPages+Hexo创建博客

发表于 2018-01-01 | 分类于 blog搭建 | 阅读次数:

在建立这个博客之前，一直没有固定的记录、写作的地方，笔记文章等弄的乱七八糟，每次找某个内容的时候总是浪费很多时间，因此建立这个博客来重新将记录过的东西重新整理一下。

个人博客的建立通常有三种选择：1.WorkPress 2.GitHubPages+JektII 3.GitHubPages+Hexo

WorkPress：需要独立域名支持，对markdown支持不友好
GitHubPages+JektII：免费，操作有些许麻烦
GitHubPages+Hexo：免费，使用简单便捷，用户众多

环境配置

在使用hexo之前首先要确保Git和Node.js已经安装好。检验两者是否已经安装的方法类似，打开terminal，分别输入：
$ git version
当有Git版本输出时，即为已经安装。

$ node -v
若有Node版本信息输入，则为已经安装。

加入没安装Git和Node.js

Git

Git用来把本地的hexo内容提交到github上去，必须安装
Xcode安装后，自带Git的。不在另外涉及。

Node.js

node用来生成静态页面，因此也必须安装。

  • node官网下载最新版本即可。
  • 利用HomeBrew安装node
    终端直接输入$ brew install node ，即可安装最新版本node

    GitHub

    用来做博客远程库、域名、服务器等，即将hexo托管在GitHub上。

    安装hexo

    Node和Git安装好之后，就可以正式安装hexo了。首先在某个目录下创建一个文件夹，取名为blog（随意取），然后cd到该目录下，安装hexo和存放hexo的配置文件。

执行如下命令安装hexo：
$ sudo npm install -g hexo

init初始化hexo：
$ hexo init

执行完毕之后，hexo已经安装成功。此时的文件夹就是博客根目录，所有的操作都要在这里面进行。

每次部署都可以分为三步：

  1. hexo clean 清除缓存，网页正常情况下可以忽略此条命令
  2. hexo generate 或者 hexo g 生成静态页面
  3. hexo deploy 或者hexo d开始部署
    这三部也可化为一步执行：
    hexo clean && hexo g && hexo d

上面的操作是直接部署到GitHubPages上，假如想要预览，那么可以启动本地服务，进行文章预览调试：
hexo server或者hexo s
浏览器中敲入 http://localhost:4000 验证

配置GitHub

这一步和以往建立GitHub项目区别很小，只是仓库名字必须为projectName.github.io，固定写法。
然后将该仓库clone到上面创建的文件夹中（blog），git clone + ssh/https

此时blog文件中有：

  1. _config.yml 配置文件，设定某个值的时候，冒号之后要紧跟一个空格。
  2. source 资源文件，其中_posts目录下存放md文件即文章，新创建的页面（page）也存在这里面，，还有解析域名的CNAME文件。
  3. themes 主题文件，自定义的hexo主题文件包存在这个目录下。

至此，GitHubPages+Hexo博客基本框架已经搭建完毕。
