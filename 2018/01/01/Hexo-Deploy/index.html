Hexo配置使用

发表于 2018-01-01 | 分类于 blog搭建 | 阅读次数:

上篇创建博客的文章已经将GitHubPages+Hexo环境搭建完毕，这篇文章主要讲Hexo的应用和配置。

_config.yml配置文件修改和设定

初次创建博客需要对Hexo配置文件进行简单的修改，例如：
首先vim _config.yml
title: 你博客的标题
subtitle: 你博客的副标题
description: 博客的描述，便于被搜索到
author: 作者名字
language: 选择语言（zh-Hans）

theme: 主题名字，在这里我选择了next，理由是精简，内容至上，使用人数多

deploy:
type: git 提交方式
repo: 创建的github仓库的https/ssh地址
branch: master 分支选择
然后执行命令npm install hexo-deployer-git --save

文章书写及提交

博客的所有文章都保存在博客文件夹下/source/_posts中，创建命令：

hexo new "title" title为自定的文章名字

执行命令后，/source/_posts中会出现以“title”命名的md文件和文件夹，md文件就是我们需要编辑的博客文章的载体，文件夹则是保存文章中所需资源（如图片）的地方。
md文件建议使用第三方工具进行编辑，简单方便，省时间。我使用的是Bear，非常好用！
文章的开头处有固定的格式：

其中title和date是创建文章时自动填写的，也可自己修改；tags和categories自己添加，也可省略不写，具体的标签和分类在相应的page中显示。这里应该还有其他的添加项，慢慢去发现。
上面提到page页面，即在自己相关信息一栏展示的几个模块。

创建新page命令（在blog目录下）：
hexo new page "title" ，执行完之后，/source下就会产生title命名的文件夹，同样的包括md文件和一个文件夹。md文件就是当前page页面的相关设定，在头部内容中加入type: "categories"或者type: "tags"再或者其他的，当写博客时填入相对应的内容就会自动分配到相关page下面。例如：我写一篇新的blog，在头部信息中加入categories: "分类一"和tags: "标签一"，那么这时在分类页面和标签页面就会分别存在“分类一”和“标签一”两个选项，两个选项中都会包含这篇blog。
注：要想把文章归到响应page，必须在page文件夹的md文件中加入type类型，文章md文件中同时写入type的值。

Hexo主题选择

Hexo原始的主题已经基本满足内容至上的要求，但是还缺乏精致感，喜欢折腾的话就给它换个主题吧。
我选择的是NexT，NexT官方文档介绍的很详细，按照步骤来就可以了。
