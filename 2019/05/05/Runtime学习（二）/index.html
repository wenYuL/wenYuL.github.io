实例对象本质

我们在初始化一个实例对象的时候通常是这样：

Object *objc = [[Object alloc]init];

那么，alloc和init分别作用是什么呢？经过查阅资料能发现，alloc的作用是系统会分配一块内存给对象，并且把地址返回给对象的isa指针，此时这个指针并不是对象最终的指针（悬挂指针），此时系统所分配的内存并没有被初始化；init的作用就是初始化一块新的内存空间给对象，此时的指针才是对象最终的指针。

根据objc源码可以看出，对象的本质就是一个结构体指针，而且只有一个isa指针，这个isa指向它对应的Class，由此可以推出对象占用的内存空间是8个字节（在64位系统下）。但是实际情况呢？系统实际分配了16个byte给对象，原因在于编译器做了内存对齐的操作。详情看本文最后部分。

那么对象在内存中是怎样存储的呢？

首先，对象一共分为三种：实例对象（instance）、类对象（class）、元类对象（meta-class）。

  • 实例对象

    通过alloc产生的对象，每一次alloc都会产生不同的实例对象；实例对象结构体中包含的信息有：1.isa指针 2.其他成员变量

那么class对象的实例方法（即类方法）存储在什么地方呢？元类对象中。

  • 元类对象.
    每个类在内存中有且只有一个meta-class对象。meta-class对象的内存结构和class对象的结构几乎一致，区别在于元类对象结构中存储的方法列表是类对象的实例方法（也就是类的类方法），而类对象中存的是对象的实例方法。

当class对象调用一个方法（即类的类方法）的时候，会根据isa的指向去meta-class中寻找，假如找不到，则根据superClass去meta-class的父类中去找，依次往上，直到找到基类（NSObject）。

补充：内存对齐

什么是内存对齐

现在计算机系统分配内存都是按照byte字节分配的，从理论上讲存储任一数据都可以从任何地址开始。但是在实际内存分配中，系统会对数据的存储地址加以限制，他会要求数据的首地址是某个数（4或者8）的整数倍，这就叫作内存对齐。

char类型占用1个byte，double类型占用8个byte，按理说这个结构体总共占用9个byte，但实际情况是16个byte。这就是内存对齐之后的结果。从输出的内存地址可以看出，char类型被分配了8个byte的存储空间。

内存对齐的原则

内存对齐有三个原则：

  1. 结构体第一个成员的偏移量（offset）为0，以后每个成员相对于结构体首地址的 offset 都是该成员大小与有效对齐值中较小那个的整数倍，如有需要编译器会在成员之间加上填充字节。
  2. 数据成员为结构体：如果结构体的数据成员还为结构体，则该数据成员的“自身长度”为其内部最大元素的大小。(struct a 里存有 struct b，b 里有char,int,double等元素，那 b “自身长度”为 8)。
  3. 结构体的总大小为 有效对齐值（对齐系数） 的整数倍，如有需要编译器会在最末一个成员之后加上填充字节。

编译器会提供一个默认的对齐系数，这个系数也可以自己通过预编译命令修改
#pragma pack(n)，n只可以在1，2，4，8，16中选择，n就是要设定的对齐系数，当设置n为1时，就是代表不进行内存对齐。XCode的编译器设置的对齐系数为8。
Windows中默认对齐数为8，Linux中默认对齐数为4。

解释：

  • 对于原则一，看上面提到的例子

char占用的空间是1个byte，double占用8个byte，如果不进行内存对齐的话，char和double的地址应该是顺序的，char被分配了1一个byte的空间；内存对齐之后，double的地址偏移量，根据double大小8byte和对齐系数8计算后得出8个byte，即偏移量为8byte，也就是double数据的地址为char的地址值加8，所以这个结构体的最终大小为16个byte。

  • 对于原则二
    遇到数据成员为结构体时，就把他看成一个整体，大小为结构体中最大数据成员的大小。
  • 对于原则三
    在计算完数据的大小后，假如大小不是对齐系数的整数倍，则在最后数据成员的内存位置添加填充字节。

为什么要进行内存对齐

尽管内存是以字节byte为单位，但大部分处理器并不是按照字节块来存取的；它一般会以双字节,4字节,8字节,16字节甚至32字节为单位来存取内存，我们将上述这些存取单位称为内存存取粒度。

现在考虑8字节存取粒度的处理器取int类型变量（64位系统），该处理器只能从地址为8的倍数的内存开始读取数据。

假如没有内存对齐机制，数据可以任意存放，现在一个int变量存放在从地址1开始的连续四个字节地址中，该处理器去取数据时，要先从0地址开始读取第一个4字节块,剔除不想要的字节（0地址）,然后从地址4开始读取下一个4字节块,同样剔除不要的数据（5，6，7地址）,最后留下的两块数据合并放入寄存器，这需要做很多工作。

因此内存对齐能提升数据存取的效率。
