Objective-C代码规范

发表于 2019-04-22 | 阅读次数:

命名规范

目录名、类名、文件名

均要首字母大写，采用驼峰式命名；有前缀的前缀也要按照驼峰式命名规则。

私有变量

私有变量在.m中声明，并以下划线_开头，开头字母小写，后边按照驼峰式命名规范书写。

属性变量

  • 命名采用小驼峰命名方式，首字母小写，后面每个单词首字母大写
  • 属性的关键字推荐按照原子性，读写，内存管理的顺序排列，并且关键字与前面分隔符之间留一个空格。
  • 相同属性修饰的变量尽量放到一起

宏和常量命名

  • 宏定义的命名字母要全部大写，单词间使用下划线_分隔开
  • 对于常量类型，在某个编译单元内的，static const修饰，名字以k或者kDefault开头；外部可访问的常量类型，仿照苹果风格，在头文件中进行extern声明，在实现文件中定义其值，采用大驼峰命名方式

enum枚举

  • Enum类型的命名与类的命名规则一致
  • Enum中枚举内容的命名需要以该Enum类型名称开头

Delegate

  • 后缀为Delegate
  • 当委托的方法过多, 可以拆分数据部分和其他逻辑部分, 数据部分用dataSource做后缀
  • 使用did和will通知Delegate已经发生的变化或将要发生的变化。

方法

  • 方法名采用小驼峰命名方式
  • 方法名不要使用new作为前缀
  • 不要使用and来连接属性参数
  • 如果方法包含多个参数时，每个参数占用一行，并以冒号对齐
  • 方法一般不使用前缀名，私有方法可使用前缀名来区分和分组
  • 对应的方法名要与参数保持高度一致
  • 表示对象行为方法、执行性的方法应以动词开头
  • 返回性的方法应以返回的内容作为开头，前面不要加get，除非是间接返回一个或多个值
